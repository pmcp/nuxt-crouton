How it works

Describe your organisation. Get a working app.

Two AI roles turn a conversation into a deployed application. You own the code — every file, every component, every database table.

01

Architect

You describe what your organisation does. The architect asks questions, understands your domain, and designs the data model — collections, fields, relationships, example data.

Outputs: schemas + seed data as JSON files
02

Designer

The designer reads each schema and decides how it should look. Simple collections get layout hints. Complex ones — like an event calendar with registration — get custom Vue components.

Outputs: layout hints on schemas or custom .vue files
03

Generate & deploy

The CLI generates everything the designer didn't: composables, types, API routes, database schema, and standard components. One click to deploy on Cloudflare's edge.

Outputs: full Nuxt app in a Git repo you own
Real output

The designer knows when to get creative

Most collections work great with standard layouts. But when the domain demands it, the designer writes custom components.

Standard and custom, same folder

Rooms are simple — a grid of cards with photos. The CLI generates that from a two-word hint. Events need a calendar, custom cards with spots-remaining badges, and an embedded registration flow. The designer writes those as real Vue components.

The app doesn't know which path produced which file. You can customise any of them.
