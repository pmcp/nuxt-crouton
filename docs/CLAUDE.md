# CLAUDE.md — `@crouton/docs` (the documentation site)

Guidance for working in `docs/`. **For all workflow, commit, issue-tracking, and
monorepo conventions, follow the root [`/CLAUDE.md`](../CLAUDE.md)** — this file
only covers what's specific to the docs app.

> Historical note: this file used to be a full (stale) clone of the old root
> guide. It was slimmed in epic #504. If you need a project-wide rule, it lives
> in the root `CLAUDE.md`, not here.

## What this is

The public documentation site for Nuxt Crouton — **live at https://nuxt-crouton.dev/**.
A Nuxt 4 app using **Nuxt Content** (Markdown) and **Nuxt UI 4**, same stack and
rules as the rest of the monorepo (`<script setup lang="ts">`, pnpm, Nuxt UI v4 only).

## Layout

| Path | What lives there |
|------|------------------|
| `content/` | The docs themselves — numbered sections (`1.getting-started`, `2.fundamentals`, …). Add/edit Markdown here. |
| `content.config.ts` | Nuxt Content collection config. |
| `app/` | App shell, layouts, components for the site. |
| `server/mcp/` | The **Docs MCP** server (`tools/`, `resources/`, `prompts/`) — see root CLAUDE.md "MCP Servers". |
| `server/api/changelogs/` + `scripts/sync-changelogs.ts` | Changelog ingestion (`pnpm --filter @crouton/docs …`). |
| `public/`, `data/` | Static assets and data. |

## Commands

```bash
pnpm --filter @crouton/docs dev        # local dev
pnpm --filter @crouton/docs build      # build
pnpm --filter @crouton/docs typecheck  # nuxt typecheck (this app)
```

## Deploy

Cloudflare **Workers** (not Pages) via `.github/workflows/deploy-docs.yml`
(`cf:deploy` → `NITRO_PRESET=cloudflare_module nuxt build` → `wrangler deploy`).
Routine deploys go to staging — see the root `/deploy` skill; never hand-roll it.

## When you change docs

- Editing `content/**` is the normal path — no package approval needed (docs is an
  app, not `packages/`).
- After changing a public API elsewhere, update the matching page under `content/`
  (the root CLAUDE.md "Maintaining AI Documentation" table points here).
- The skills-overview HTML (`writeups/architecture/skills-and-triggers.html`) is
  generated by the **root** `scripts/gen-skills-doc.mjs`, not from here.
