Maquette — what the build exports 0 JS · static

The reactive builder (Vue + reka-ui) is the design-time tool. What ships is this: a layout that reflows entirely in CSS container queries — the authored breakpoints become @container rules. No framework, no reactivity, no runtime. The preview below is driven only by the stylesheet shown next to it.

Mode A — nuxt-maquette (Nuxt module)

Dev loads the full reactive builder. Build clashes it down to the layout you decided on → static CSS + markup injected into the page. Production ships no layout JS.

Mode B — @fyit/maquette / vue-maquette (compiler)

The same compileLayoutToCss(tree) exposed standalone: a saved layout → a .css file (+ optional HTML skeleton). Any stack. This is the library face.

…emitted as this

/* maquette.css — generated from the saved LayoutTree. No JS. */
.mq-app {
  display: grid;
  gap: 8px;
  grid-template-columns: 1fr;            /* phone-first */
  grid-template-areas: "list" "main" "stats";
}
/* authored breakpoint @ 640px → master-detail */
@container mq (min-width: 640px) {
  .mq-app {
    grid-template-columns: 200px 1fr;
    grid-template-areas: "list main" "stats main";
  }
}
/* authored breakpoint @ 960px → + stats column */
@container mq (min-width: 960px) {
  .mq-app {
    grid-template-columns: 200px 1fr 220px;
    grid-template-areas: "list main stats";
  }
}
<!-- maquette.html — minimal skeleton, your blocks slot in -->
<div class="mq-root">
  <div class="mq-app">
    <section class="mq-list">…</section>
    <section class="mq-main">…</section>
    <section class="mq-stats">…</section>
  </div>
</div>

Why two modes is the win: Mode A is just Mode B + a thin Nuxt wrapper — so building the compiler once gives you both a great Nuxt DX and a framework-agnostic OSS library. The authored breakpoints (min-width, lock-upward) map 1:1 onto @container rules; a snap-composed split tree maps onto grid-template-areas; the viability metric becomes a build-time check. Collapse/overlay behaviours that genuinely need interaction can emit a tiny optional JS shim, but the layout itself is pure CSS.
