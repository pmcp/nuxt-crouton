Idea lands
No code yet. The issue exists before the code does.
🛠 github-tasks — opens the epic + sub-issues, labelled by package/app, written as a hypothesis.
🛠 ecosystem-check (optional) — does Nuxt/UnJS already solve it?
⚙️ decompose-on-issue.yml — fires on issues: opened/labeled.
⚑ Hard gate — editing files with no issue? Stop and open it first.
Hand it to agents instead: 👤 /task-decompose "build X" → orchestrator → decomposer → worker.
1
Sign-off gates
Agree on the shape before writing code. Each holds on status:blocked.
🛠 schema-review — data model → field table (#314)
🛠 ui-proposal — a .vue/layout/theme → live preview (#307)
🛠 test-review — logic in packages/* → failing test first (#774)
⚙️ resume-on-comment.yml — your reply unblocks the pipeline.
⚑ Only a comment lgtm/approve resumes it — a 👍 reaction or a label does nothing (#572).
2
Build it
All local — no CI yet.
🛠 crouton — schema → Form/List/API/migration, arranged into a default layout.
🛠 db-migrations — only at the migration step.
pnpm typecheck after every change. Not optional.
⚑ A hook blocks edits to packages/* without approval. New apps scaffold into pocs/, not apps/.
3
Commit
The first automated quality pass.
👤 /commit — never raw git commit.
🛠 sync-docs + 🛠 i18n-check run first, then granular conventional commits referencing (#NN).
4
PR opens · CI fans out
Most CI lives here. PR body carries Closes #NN.
⚙️ ci.yml — typecheck + lint + test
⚙️ e2e.yml — boot → auth → CRUD on a real app
⚙️ frontend-review.yml — Nuxt UI 4 conventions (on .vue)
⚙️ a11y.yml · red-team.yml — accessibility + security probes
⚙️ deploy-pocs.yml — a clickable preview URL (pocs/)
⚙️ pipeline-pr-status.yml — posts the status note
5
The robot-PR safety net
When an agent finishes a task — a change never ships unseen.
1. Robot finishes → opens PR
2. Tests run → you always get a status note
3a. All green → ⚙️ automerge-epic-subpr.yml merges it
3b. Fails → ⚙️ fix-ci-on-failure.yml tries a fix … still stuck → tags you
4. Breaks after merge → opens a new ticket
6
Merge to main = staging
Trunk = staging (#347). Landing code previews itself.
⚙️ deploy-app/-apps/-pocs.yml → <app>.pmcp.dev
⚙️ cleanup-merged-branches.yml — tidies branches.
⚑ Structurally can't reach production — a push never sets environment=production.
7
Production
Deliberate, human-only. Never a side effect of an agent flow (#318).
👤 /deploy-production → <app>.friendlyinter.net via manual dispatch.
A POC must be promoted pocs/ → apps/ first.
8
Close & learn
🛠 postmortem — at epic close, mints workflow issues so the loop tightens.
⚙️ epic-digest · housekeeping — scheduled digests, no LLM.
Ongoing: 🛠 dependency-sweep · e2e-smoke · db-clone
The whole thing in one breath
idea → github-tasks (issue first) → sign-off gates (hold on lgtm) → crouton build + typecheck → /commit → PR → CI fans out → status note → green merges / fail fixes → merge = staging deploy → /deploy-production (humans only) → postmortem.
Burn in two things: the issue exists before code, and an lgtm/approve comment — not a reaction, not a label — is the only thing that unblocks a held pipeline.
