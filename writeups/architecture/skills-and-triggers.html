Skills — what fires when

Every skill in .claude/skills/, grouped by job, with the trigger that fires it. Three kinds of trigger:

Automatic — a hook / pre-commit step runs it for you On ask — you invoke it (/name) or request the work In flow — fires inside the agent task pipeline

1 Build & generate

Turn intent into code (or into issues that become code).

/croutonon ask

Fires when: you generate a CRUD collection — schema JSON → Form/List/API/composable/schema.

The collection generator workflow (CLI crouton config). The backbone of building an app's data surfaces.

/ui-proposalin flowon ask

Fires when: a task adds/changes UI (a .vue, a layout, a screen), or you ask to “mock this up”.

Builds a before/after HTML mockup (offline) + renders a PNG for design sign-off before the real component is built. (Epic #307.)

/task-decomposeon askin flow

Fires when: “break this down into issues and work it” / “spawn agents to build X”.

One task → an epic + recursive tree of sub-issues → orchestrator → decomposer → worker agents.

2 Plan & track

Before code exists — the ISSUE-FIRST gate and prior-art checks.

/github-taskson askin flow

Fires when: planning a feature, “make issues”, “track this in GitHub” — and the ISSUE-FIRST hard gate before any new initiative.

Creates epics + labelled sub-issues (pkg:/app:/meta:), tied to the commit/PR workflow.

/ecosystem-checkon ask

Fires when: weighing build-vs-adopt or starting new infra/tooling.

Checks Nuxt / UnJS / Vite / OSS prior art first (it’s often already solved) within the project’s constraints.

/mcp-ideaon ask

Fires when: a task reveals repetitive work an MCP tool could automate.

Drops the idea onto the MCP backlog (.claude/mcp-ideas.md).

3 Commit gates

These run automatically as part of landing a change — you rarely call them by hand.

/sync-docsauto · pre-commit

Fires when: automatically before every /commit.

Updates only the docs the change touches (package CLAUDE.md, docs pages, skills). Assumes docs are stale; never over-reaches.

/i18n-checkauto · pre-commit

Fires when: committing and the diff includes .vue files in an i18n app (or as a review add-on).

Catches hardcoded labels, missing locale keys, locale-sync issues — fast, only what changed.

/commiton askrequired

Fires when: every commit — mandated for all commits (never git commit directly).

Granular, conventional commits; filters to session-relevant files, groups by intent, runs sync-docs/i18n-check first.

4 Review & audit

Quality checks on demand.

/reviewon ask

Fires when: reviewing uncommitted changes or recent commits, before or after committing.

Pattern violations, security issues, bugs — tuned for a solo dev moving fast.

/auditon ask

Fires when: checking package docs / documentation health across the monorepo.

Detects drift between code and docs; reports completeness.

/sync-checkon ask

Fires when: verifying field-type sync across generator ↔ MCP ↔ skill ↔ docs.

Reports mismatches with fix instructions (the CLI’s mandatory sync workflow).

5 Deploy

Code → a live Cloudflare Workers URL.

/deployon ask

Fires when: deploying a launched app in apps/ (first bootstrap, CI wiring, or routine deploy; or migrating a Pages app).

Auto-provisions D1+KV, syncs ids, migrates, two-domain staging/prod.

/poc-deployon ask

Fires when: giving a pocs/ app a staging preview URL (or “build X as a poc” needs a clickable link).

The lighter, staging-only POC path — every PR deploys an isolated preview.

6 Verify & maintain

Prove nothing broke; keep the repo current.

/e2e-smokeon ask

Fires when: after a packages/ change or a dependency bump — “smoke test”, “run e2e”.

Boots a real generated fixture app → login → CRUD → package surfaces (Playwright).

/dependency-sweepon askscheduled

Fires when: “update dependencies” / “are we on the latest”, or the recurring quarterly sweep ticket comes due.

Sweep → triage (safe/deliberate/wait) → bump the pnpm catalog → typecheck + e2e gate. (No update bot by design.)

/db-migrationson ask

Fires when: a schema change needs a Drizzle migration generated/applied (the schema.mjs-after-build gotcha), or adding a package-owned infra table.

The migration step — not the normal collection flow (that’s /crouton).

/session-start-hookauto · session starton ask

Fires when: setting up a repo for Claude Code on the web — the SessionStart hook that boots the env each session.

Ensures the project can install / run tests / lint inside web sessions.

Triggers reflect the skill descriptions + CLAUDE.md rules (ISSUE-FIRST, “always /commit”, sync-docs runs before commit). Agents (task-orchestrator → task-decomposer → task-worker) drive the “in flow” skills. Inline SVG · no JS · renders offline.
