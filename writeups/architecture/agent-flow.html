How one task becomes shipped code

A request like /task-decompose "build X" doesn't go to one agent — it fans out into a tree of GitHub issues, and each leaf is built, sign-off-gated, and turned into a PR by its own worker. This is that flow in one frame: the fan-out, and the four moments the work surfaces back on the ticket.

Highlighted = a moment a human is in the loop. Everything else runs on its own.

Every node runs the LEAF TEST. A single coherent change with bounded files, clear acceptance, doable in one focused run → build it (spawn a worker). Otherwise → split it into sub-issues and recurse. Hard stops keep it from running away: MAX_DEPTH = 3, MAX_CHILDREN = 6. Orchestrator and decomposer only read/write issues, so they run on Sonnet; only the worker writes real code, on Opus.

▸ Inside one worker — a leaf, end to end

Each leaf runs in its own git worktree, off the shared epic/<NN>-slug branch so it sees its siblings' merged work.

Claim the issue

Reads the issue + its epic, sets the board moving.

label status:in-progress

Prerequisite & invariant check — block, don't improviseasks for help

If something a sibling issue was meant to create isn't there yet, it does not scaffold a second copy — it stops and asks. (The #325 post-mortem fix: parallel workers once built three conflicting copies of one package.)

comment + @pmcp + status:blocked → STOP
3

Branch off the epic branch

One issue → one branch → one PR.

claude/issue-<N>-<slug>
4

Sign-off gate — propose before you buildyou approve

Changing a screen or a data model? It mocks it first — a UI mockup or a schema field-table — posts it on a draft PR, and holds. You review on the committed .md in the diff; it resumes on lgtm / 👍 / ui-approved.

ui-proposal · schema-review → draft PR + status:blocked
5

Implement → typecheck → commit

Builds it, makes typecheck green (non-negotiable), commits in small atomic pieces, pushes immediately.

pnpm typecheck · /commit (no git add .) · never squash
6

Open the PR — into the epic branch

Body carries Closes #NN so the issue auto-closes on merge; one epic→main PR is the single human review.

create_pull_request · base: epic/<NN>-slug
7

Prove the mockup became realityscreenshot

After an approved UI build, it captures a real before/after of the changed surface and posts it on the PR.

scripts/app-shots.mjs → before/after on the PR (#311)
8

Hand back a clickable previewpreview link

For a POC, CI auto-provisions and deploys a staging Worker, then posts the auth-working URL on the PR — a link you can log into, not just code on a branch.

poc-deploy → https://<name>.pmcp.dev on the PR

★ The four moments it reaches you

The agents are headless, so they never pop a dialog. Everything that needs a human happens as a comment, label, or attachment on the GitHub issue/PR — which is exactly why this surface ports cleanly (Linear can mirror it, Slack can notify on it, pi can post to it).

Asks for help — never guesses

On a real blocker, it comments what's blocking, @mentions you, and marks the ticket blocked. Your reply resumes it — no special command.

add_issue_comment + @pmcp + status:blocked

Shows you a change before building it

UI and schema changes get a rendered mockup / field-table on a draft PR. You give feedback inline on the committed .md — pinned to the exact line — and approve when it's right.

sticky comment · lgtm / 👍 / ui-approved

Proves it built what it showed

After the build, a real before/after screenshot of the surface lands on the PR — no checking out the branch to see it.

app-shots.mjs · separate sticky comment

Hands back a link you can click

Every POC PR deploys an isolated staging environment and posts a working, loggable-in URL — the deliverable is a preview, not a diff.

https://<name>.pmcp.dev · auth seeded

⛨ What keeps the fan-out safe

The pipeline can run unattended, so the guardrails are structural, not vibes.

epic branch siblings integrate before main — no duplicate scaffolding wave-gating foundation issues run before the work that depends on them packages/ HARD GATE a hook blocks shared-code edits unless the epic is approved no-squash atomic, blameable commits land on main — built for later archaeology block, don't improvise a missing prerequisite is a STOP, never a guess
Today vs. nearly-there. The fan-out, sign-off gates, help-pings, screenshots, and preview links above are live. The fully-autonomous watch-to-merged loop — sticky CI status, auto-merge on green, a fix-bot on red — is partially built (epic #336); until it lands, an attended session babysits the PR to merge.
