…and the loops with no human in them
🔧 Autofix — ⚙️ fix-ci-on-failure.yml on CI red → a bot fixes & re-pushes. Capped at 3; never touches packages/; at the cap it 🛑 tags you.
🔀 Auto-merge — ⚙️ automerge-epic-subpr.yml: a green, pipeline-authored sub-PR onto an epic/ branch merges itself.
🌊 Waves — ⚙️ schedule-waves.yml: Blocked-by: #NN edges; when all blockers close it delegates the next worker.
💬 Comment dispatch — you comment 👤 /delegate or /deploy → ⚙️ comment-dispatch.yml picks it up, reacts 🚀.
📊 Status note — ⚙️ pipeline-pr-status.yml: a sticky PR + epic roll-up, refreshed on every check-suite.
The whole thing in one breath
idea → 🛠 github-tasks (issue first) → gates that post 🟡, take your command 🟢, fire an auto-comment 🟣 and re-post until lgtm → 🛠 crouton build → 👤 /commit → ⚙️ CI (autofix / automerge run themselves) → merge = staging → 👤 /deploy-production → 🛠 postmortem.
Burn in: the issue exists before code, and an lgtm/approve comment — not a reaction, not a label — is the only thing that unblocks a held gate.
