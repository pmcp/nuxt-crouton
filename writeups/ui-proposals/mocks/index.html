📐 Responsive — per container & per device

18Open ▸

Container Context

⭐ the real model

Every layout carries its own size context: each region reflows to its own width, not the screen's — real CSS @container queries. Drag a seam and one region becomes a 3-col grid while its neighbour is a list; resize the whole frame and the calendar flips from a month grid to an agenda. The block-authoring rule (size to the pane, never the viewport) made recursive. Drag the corner & the seams.

@container · recursive · epic #855
19Open ▸

Breakpoint Authoring

⭐ author by demo

Set breakpoints by demonstration: drag the width, arrange the panes, and that arrangement locks from that width up (a min-width checkpoint on the ruler). Drag wider, rearrange, lock another. Then sweep the width and the layout snaps between the checkpoints you set. Each widget can also switch its own variant per breakpoint (Bookings: list → cards → table) and a pane can be collapsed on a given breakpoint (it docks as a gutter tab — tabs/rail style). Hover the handle for device-size shortcuts. The visual way to author responsive breakpoints.

checkpoints · min-width · epic #855

Responsive · per device

⭐

Toggle the frame to Desktop / Tablet / Phone / Square and the layout reroutes to that screen's saved arrangement. Arrange each device differently (grip to rearrange, seam to resize) — every edit is saved per device (localStorage), so it jumps back when you switch. This is how you author responsiveness.
