Nuxt Crouton is a **modular ecosystem** consisting of 14 packages (1 core, 1 generator CLI, 1 auth layer, 1 themes package, and 10 addon layers). This architecture provides flexibility, keeps bundle sizes small, and allows you to use only what you need.

::callout{icon="i-heroicons-information-circle" color="blue"}
**Single-install experience:** Installing `@fyit/crouton` automatically includes the **i18n**, **auth**, and **admin** packages. You don't need to install them separately.
::

## Core Packages

### 1. @fyit/crouton

**Purpose**: Core runtime library with auto-included auth, admin, and i18n
**Install**: `pnpm add @fyit/crouton`

**Auto-includes**:
- `@nuxthub/core` - Database (D1/SQLite), KV storage, blob storage
- `@fyit/crouton-i18n` - Translation system
- `@fyit/crouton-auth` - Authentication with teams
- `@fyit/crouton-admin` - Super admin dashboard

**Contains**:
- Composables (useCollectionQuery, useCollectionMutation, useCrouton, useCroutonApps)
- Base components (CroutonList, CroutonButton, CroutonForm)
- Modal management
- Cache invalidation system
- App auto-discovery system
- TypeScript types

**When to use**:
- Required for all Nuxt Crouton projects
- This is the foundation everything else builds on

**Config**:
```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  extends: [
    '@fyit/crouton'  // Includes auth, admin, i18n
  ]
})
```

### 2. @fyit/crouton-cli

**Purpose**: CLI tool for code generation
**Install**: `npm install -g @fyit/crouton-cli`

**Contains**:
- CLI commands (crouton-generate, crouton-rollback)
- Code generators
- Template system
- Schema validation

**When to use**:
- During development to generate collections
- Not needed in production (generated code doesn't depend on it)

**Commands**:
```bash
crouton-generate <layer> <collection>
crouton-rollback <layer> <collection>
crouton-rollback-bulk --layer=<name>
crouton-rollback-interactive
crouton-generate init
crouton-generate install
```

**Key insight**: This is a **dev-time dependency**, not a runtime dependency. Once code is generated, your app doesn't need this package.

### 5. @fyit/crouton-supersaas

**Purpose**: SuperSaaS integration layer - connectors, app-level translations, and utilities

**When to use**:
- Building apps with SuperSaaS for team/user management

### 12. @fyit/crouton-auth (Auto-Included)

**Purpose**: Authentication layer with teams, passkeys, and 2FA
**Install**: Auto-included with `@fyit/crouton`

**Contains**:
- Better Auth integration
- Team/organization management (multi-tenant, single-tenant, personal modes)
- Passkey/WebAuthn support
- Two-factor authentication (TOTP)
- OAuth providers (Google, GitHub, etc.)
- Auth composables (useAuth, useSession, useTeam)
- Pre-built auth components (forms, OAuth buttons, 2FA setup)

**Routes provided**: `/auth/*` (login, register, forgot-password, etc.)

**Provides translations**:
- `auth.*` - Authentication strings (signIn, register...)
- `account.*` - Account settings strings
- `teams.*` - Team management strings

**Config** (optional customization):
```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  extends: ['@fyit/crouton'],

  croutonAuth: {
    teams: {
      allowCreate: true,    // Can create teams (SaaS pattern)
      showSwitcher: true    // Show team switcher UI
    },
    emailPassword: true,
    passkeys: true,
    twoFactor: true,
    oauth: {
      google: true,
      github: true
    }
  }
})
```

**Environment Variables**:
```bash
BETTER_AUTH_SECRET=your-secret
BETTER_AUTH_URL=http://localhost:3000
GOOGLE_CLIENT_ID=...
GITHUB_CLIENT_ID=...
```

**Note**: This package is automatically included when you install the core package. No separate installation needed.

### 13. @fyit/crouton-admin (Auto-Included)

**Purpose**: Super admin dashboard for system-wide user and team management
**Install**: Auto-included with `@fyit/crouton`

**Contains**:
- Super admin dashboard with stats
- User management (list, create, ban, unban, delete)
- Team oversight (view all teams/members)
- User impersonation (debug as any user)
- Super admin middleware
- Admin composables (useAdminUsers, useAdminTeams, useAdminStats, useImpersonation)

**Routes provided**: `/super-admin/*`

**Provides translations**:
- `superAdmin.*` - Super admin dashboard strings

**Pages provided**:
- `/super-admin` - Dashboard with stats overview
- `/super-admin/users` - User management
- `/super-admin/teams` - Team oversight

**Config** (optional customization):
```typescript
// nuxt.config.ts
export default defineNuxtConfig({

    public: {
      crouton: {
        admin: {
          routePrefix: '/super-admin',
          impersonation: true,
          statsRefreshInterval: 30000
        }
      }
    }
  }
})
```

**Middleware**:
```vue
<script setup>
definePageMeta({
  middleware: 'super-admin'  // Protect super admin routes
})
</script>
```

**Note**: This package is automatically included when you install the core package. No separate installation needed.

### Three-Tier Route Architecture

| Tier | Route Pattern | Purpose | Access |
|------|---------------|---------|--------|
| **User** | `/dashboard/[team]/*` | User-facing features | Any team member |
| **Admin** | `/admin/[team]/*` | Team management | Team admins/owners |
| **Super Admin** | `/super-admin/*` | System management | App owner only |

## Why This Architecture?

### âœ… Benefits

**1. Small Bundle Sizes**
- Only include what you need
- No unused translation code if you don't need i18n
- No Tiptap bundle if you don't need rich text

**2. Independent Updates**
- Core can be updated without breaking addons
- Addons can evolve independently
- Generator improvements don't require core updates

**3. Clear Separation**
- Generated code vs runtime code
- Core features vs optional addons
- Development tools vs production dependencies

**4. Flexible Adoption**
- Start simple, add complexity as needed
- Remove addons you're not using
- Mix and match features

### ğŸ¯ Design Decisions

**Q: Why is the generator a separate package?**
A: It's a dev-time tool. Your production app doesn't need the generator, templates, or CLI code. Keeping it separate reduces bundle size.

**Q: Why are i18n, editor, and supersaas separate?**
A: Not every app needs translations, rich text, or SuperSaaS integration. By making them optional, we keep the core package lean.

**Q: Can I use the core without the generator?**
A: Yes! You can write components manually and still use the core composables and utilities.

**Q: Do addons depend on each other?**
A: No. Each addon only depends on the core. You can use i18n without editor or supersaas, and vice versa.

## Field Types and Required Packages

When defining collection schemas, certain field types require additional packages to be installed. The generator does **not** automatically install these dependenciesâ€”you must install them manually.

### Field Type â†’ Package Mapping

| Field Type | Required Package | Install Command |
|------------|------------------|-----------------|
| `editor` | `@fyit/crouton-editor` | `pnpm add @fyit/crouton-editor` |
| `i18n` / `translation` | `@fyit/crouton-i18n` | `pnpm add @fyit/crouton-i18n` |
| `asset` (refTarget: assets) | `@fyit/crouton-assets` | `pnpm add @fyit/crouton-assets` |
| `map` / `location` | `@fyit/crouton-maps` | `pnpm add @fyit/crouton-maps` |
| `flow` | `@fyit/crouton-flow` | `pnpm add @fyit/crouton-flow` |

### Example

If your schema uses an `editor` field type:

```json [product-schema.json]
[
  { "name": "title", "type": "string" },
  { "name": "content", "type": "editor" }
]
```

You must install the editor package:

```bash
pnpm add @fyit/crouton-editor
```

And add it to your `nuxt.config.ts`:

```typescript [nuxt.config.ts]
export default defineNuxtConfig({
  extends: [
    '@fyit/crouton',
    '@fyit/crouton-editor'  // Required for 'editor' field type
  ]
})
```

**Tip**: Check your schema files for special field types before running the generator. Install all required packages first to avoid runtime errors.

## Package Versions

Nuxt Crouton follows semantic versioning:

**Core package version**: 1.2.0
- Major: Breaking changes
- Minor: New features (backward compatible)
- Patch: Bug fixes

**Addon packages**: Independent versions
- Each addon can update independently
- Core version compatibility specified in peer dependencies

## Troubleshooting

### "Module not found: @fyit/crouton"

The core package isn't installed:
```bash
pnpm add @fyit/crouton
```

### "TranslationsInput is not defined"

The i18n addon isn't installed:
```bash
pnpm add @fyit/crouton-i18n
```

Then add to nuxt.config.ts extends.

### "EditorSimple is not defined"

The editor addon isn't installed:
```bash
pnpm add @fyit/crouton-editor
```

Then add to nuxt.config.ts extends.

### "crouton-generate command not found"

The generator isn't installed globally:
```bash
npm install -g @fyit/crouton-cli
```

### Peer dependency warnings

Install the required peer dependencies:
```bash
pnpm add @nuxt/ui @nuxt/icon
```
